# Fortune 1000 Public API / Developer Portals

Source data: `0-working/fortune/fortune_portal_master.csv`. Generated 2026-05-11 from the api-evangelist network.

Combined evaluation of (1) apis.yml type + URL pattern scan across all 996 Fortune-tagged providers, and (2) web verification of the 620 that had no apis.yml evidence.

## Summary

| Tier | Has Portal | Total | Rate |
|---|---:|---:|---:|
| **Fortune 100** | **83** | 100 | **83.0%** |
| **Fortune 500 (101-500)** | **205** | 400 | **51.2%** |
| **Fortune 1000 (501-1000)** | **203** | 496 | **40.9%** |
| **Combined** | **491** | 996 | **49.3%** |

## Methodology

Three signals combined:
1. **`apis.yml_strong`** — provider's `apis.yml` has a property typed `DeveloperPortal`, `Portal`, `APIPortal`, or `DataPortal` (293 companies)
2. **`apis.yml_medium`** — apis.yml lists a portal-shaped URL (host matching `developer.*`, `api.*`, `opendata.*`, `*.portal.azure-api.net`, etc.) (83 companies)
3. **`web_verified`** — web-probed common subdomains and ran a targeted web search for each remaining Fortune company without apis.yml evidence (115 companies)

**Companies with NO portal found (505):** confirmed via subdomain probing + 1 web search each. Mostly industrials, energy, REITs, holding companies, and B2C retailers without public developer programs.
